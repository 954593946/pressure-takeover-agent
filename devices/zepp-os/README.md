# Zepp OS 路线

目标设备：Amazfit Active 2。该路线验证消费级腕上形态，P0 包括自研应用真机运行、蓝/黄/绿状态、短文本、真实震动和手机 Side Service/网关通信。

建议两到三天内验证：

- 能否安装/预览自研应用。
- 能否稳定显示状态页和触发震动。
- 能否从手机接收状态更新。
- 可读取哪些健康数据；读取失败不得阻塞主流程。

若安装、震动、状态显示或通信任一主链路无法稳定，应按项目止损规则切换 ESP32-S3 为主 Demo 设备，同时保留 Zepp OS 探索记录。

对外边界：设备或 Side Service 将本地协议转换为 `contracts/` 中的事件与 World State 子集。
