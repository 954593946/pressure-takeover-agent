# Agent 提示词

提示词应像代码一样版本化和评审。建议按用途拆分：任务解析、接管决策、消息草稿、车内短话术；不要把所有任务塞进一个超长 system prompt。

每个提示词必须记录：输入 Schema、输出 Schema、允许/禁止行为、固定测试样例、模型与 temperature。核心决策要求同一场景 10/10 一致，失败时回退到规则或固定模板。

最低安全要求：不贴情绪标签、不做医疗判断、不将心率作为单点依据、不越权发送、不建议加速抢时间、驾驶中一次只给一个判断和一个确认。

## 当前文件

| 文件 | 用途 |
| --- | --- |
| `task_parser_v1.md` | 手机端语音任务解析，识别刚性责任和弹性任务。 |
| `message_draft_v1.md` | 生成老师和家人消息草稿，等待用户确认。 |
| `in_car_briefing_v1.md` | 生成车机一句话现实结论和确认提示。 |
| `order_preference_v1.md` | 生成超市任务后置后的模拟配送方案摘要。 |
| `test_cases.md` | P0 主链路、安全约束和回归检查集。 |

这些文件是评审材料和后续 Agent LLM 接入依据；当前后端仍可继续使用规则兜底，避免 Demo 受模型不稳定影响。
