# AURI 六周 Demo 范围

> 开发执行以根目录《随行压力接管Agent_开发功能清单_v1_五人分工版.docx》为主，本文件用于快速判断一项工作是否属于本轮范围。

## 固定场景与目标

用户在车外通过手机创建“18:10 接孩子，之后去超市”。会议延迟和拥堵使 ETA 变为 18:28。AURI 识别接孩子为刚性责任、超市为可替代弹性任务；上车后把主交互端切到车机，进入 L2 协调接管，生成老师/家人消息和基于用户 Profile 的线上采购方案。用户只做一次安全确认，系统模拟消息发送和下单，三端同步完成并进入 cooldown；停车后手机展示完整复盘。

## P0 验证命题

1. 责任风险：能稳定区分 L0/L1/L2/L3，输出可解释 `reason_codes`，不做医学或情绪诊断。
2. 交互所有权：车外手机、车内车机、腕上低干扰辅助；任一时刻只有一个可操作确认入口。
3. 现实行动：不止提醒或后置任务，而是把超市任务推进为可确认的 `order_preview`。
4. 个性化：Profile 改变话术、触觉、预算、配送和替代规则，但不改变安全权限。
5. 安全执行：动作权限、合并/拆分确认、幂等、旧入口失效和 Action Ledger 真实运行。

## 必须真实运行

- 手机业务 UI、连接状态、语音/文本兜底和停车后复盘。
- 车机 HMI 的状态渲染、输出预算、单一确认和实时同步。
- 腕上真设备显示、真实震动、状态命令与 ACK；心率不是主链路。
- Agent World State、事件、任务解析、L0-L3、交互调度、Profile、动作编排、权限与幂等。
- 订单预览、Mock Adapter 接口、唯一 `order_id`、状态广播和 Ledger。
- 控制台事件注入、故障模式、日志和一键重置。

## 本轮允许模拟

- 会议延迟、接近/进入车辆、路况、ETA、驾驶行为和部分腕上信号。
- 商品、库存、价格、配送窗口、消息发送和下单结果。
- 可选车内视觉只允许结构化特征，不采集或保存原始人脸视频。

## 本轮明确不做

- 真实车辆/CAN/量产车机 SDK、车辆控制、完整导航。
- 真实微信/短信、电商账号、OAuth、支付和受限商品。
- 医疗诊断、单点生理信号推断情绪、自动猜测人格。
- 腕上长对话、商品明细、复杂编辑或导航。
- 通用语音助手、开放式陪聊、多场景泛化。

## 核心验收数字

- 标准语音 10 次至少成功 9 次；固定核心字段和 L0-L3 决策 10/10 一致。
- Agent 输出、服务适配器响应等 Schema 样例 50/50 通过。
- 三端状态与 `primary_surface` 在 1 秒内一致；语音到响应小于 3 秒。
- 按钮双击 + 语音并发、同一确认重复 10 次，只生成一个动作结果和一个 `order_id`。
- 完整脚本 10 次至少 9 次无需人工改代码或状态；汇报前完整彩排至少 3 次。
- LLM、ASR、WebSocket、BLE、服务缺货和超预算均有明确降级。
