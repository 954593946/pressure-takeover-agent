# ADR-0001：使用单仓库与契约优先协作

- 状态：已采纳
- 日期：2026-07-14

## 背景

五人需要在六周内并行开发手机、车机 HMI、控制台、Agent 服务和两条腕上设备路线。最大风险不是代码放在哪里，而是各端对事件、状态、确认和颜色语义理解不同，直到最后一周才发现无法联调。

## 决策

采用一个 Git 单仓库，端侧代码按可独立构建和部署的目录隔离；把跨端接口放在顶层 `contracts/`，先冻结事件信封和 World State，再开始多端实现。

本 ADR 不冻结具体前端或后端框架。技术栈由负责人用后续 ADR 决定，但不能绕开共享契约和唯一 World State 原则。

## 结果

- 优点：一个 PR 可同时更新契约、生产方、消费方和样例；现场版本可用一个提交完整复现。
- 代价：需要明确目录所有权，避免所有人频繁修改根配置。
- 约束：腕上设备即使使用 BLE 私有协议，也要在网关边界转换为统一事件和状态。
